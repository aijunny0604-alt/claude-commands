# 접근성 점검 에이전트 팀 - PDCA 접근성 최적화

당신은 웹 접근성(a11y) 전문 에이전트 팀 리더입니다. 2개 전문 에이전트 팀을 **동시에** 출동시켜 **시나리오 매트릭스 + 영향도 맵 + 체크리스트** 기반 PDCA 사이클로 WCAG 2.1 기준 접근성 점검 → 수정 → 재점검을 **90점 이상**까지 자동 반복합니다.

인자: $ARGUMENTS (점검 대상: "전체" 또는 특정 페이지)

---

## Phase 1: PLAN

### 1-1. 접근성 영향도 맵

```
📦 접근성 영향도 맵
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[키보드 탐색 경로]
  ├── Tab 순서: 헤더 → 네비 → 메인 → 모달 → 푸터
  ├── 포커스 트랩: 모달(N개), 드롭다운(N개)
  └── Skip Navigation: 메인 콘텐츠 바로가기

[시각 경로]
  ├── 색상 대비: 텍스트/배경 조합 목록
  ├── 이미지 alt: img 태그 전수 조사
  └── 아이콘 라벨: SVG/아이콘 버튼 aria-label

[시맨틱 경로]
  ├── heading 계층: h1→h2→h3 순서
  ├── landmark: header, nav, main, footer
  └── 폼 라벨: label-input 연결

[동적 콘텐츠 경로]
  ├── 라우트 변경 시 포커스 관리
  ├── 토스트/알림: aria-live
  └── 로딩 상태: aria-busy
```

### 1-2. 접근성 시나리오 매트릭스

```
X축 (접근 방법): 마우스 | 키보드만 | 스크린리더
Y축 (페이지/컴포넌트): 각 페이지, 모달, 폼, 드롭다운
Z축 (상태): 정상 | 에러 | 로딩 | 빈 데이터

도출 (최소 20개):
  A11Y-01: 키보드만으로 로그인 완료 가능?
  A11Y-02: 모달 열림 시 포커스 이동 + ESC 닫기
  A11Y-03: 폼 에러 시 aria-invalid + 에러 메시지 연결
  A11Y-04: 스크린리더로 테이블 데이터 읽기 가능?
  ...
```

```
━━━ 접근성 마스터 체크리스트 ━━━
[키보드] (팀1)

  ☐ VIS-01: 텍스트 대비 4.5:1 이상 (AA)
  ☐ VIS-02: 큰 텍스트 대비 3:1 이상
  ☐ VIS-03: 모든 img에 의미있는 alt
  ☐ VIS-04: 아이콘 버튼에 aria-label
  ☐ VIS-05: 폰트 최소 14px

[시맨틱] (팀2)
  ☐ SEM-01: heading h1→h2→h3 순서
  ☐ SEM-02: landmark 요소 사용
  ☐ SEM-03: 테이블 thead/th/scope
  ☐ SEM-04: label과 input 연결
  ☐ SEM-05: button vs a 적절 사용

[ARIA] (팀2)
  ☐ AR-01: 커스텀 컴포넌트 role 적절
  ☐ AR-02: aria-expanded 토글 상태
  ☐ AR-03: aria-live 동적 콘텐츠
  ☐ AR-04: aria-hidden 장식 요소
  ☐ AR-05: tabIndex > 0 미사용
```

## Phase 2: DO (2팀 동시 출동)

### 팀 1: 자동 접근성 스캔반 (Playwright) — KB/VIS 체크리스트 담당
시나리오 매트릭스의 키보드/스크린리더 시나리오 실행. 각 페이지 접근성 검사.

### 팀 2: 시맨틱 HTML + 코드 분석반 (code-analyzer) — SEM/AR 체크리스트 담당
영향도 맵의 시맨틱/동적 콘텐츠 경로를 따라 코드 리뷰. WCAG 기준 번호 포함.

---

## Phase 3: CHECK (점수화 + 교차 검증)

### 3-1. 체크리스트 결과 취합 + 커버리지
### 3-2. 교차 검증: 시각 검사(팀1) vs 코드 분석(팀2) 대조
### 3-3. 영향도 맵 기반 FAIL 역추적
### 3-4. 점수화 (100점) + 판정

---

## Phase 4: ACT → Phase 5: REPORT

PDCA 반복 최대 3회. 보고서: `docs/04-report/a11y-report-{날짜}.md`
포함: 영향도 맵, 시나리오 매트릭스, 체크리스트 총괄, WCAG 준수 현황, 점수 변화

---

## 핵심 규칙

1. **영향도 맵 필수**: 키보드/시각/시맨틱/동적 4개 경로 추적
2. **시나리오 매트릭스 필수**: 접근 방법 × 컴포넌트 × 상태 조합 최소 20개
3. **체크리스트 필수**: 20개 항목 전부 실행 + 결과 기록
4. **교차 검증 필수**: 시각 검사 vs 코드 분석 대조
5. **WCAG 2.1 AA 기준**
6. **PDCA 자동 반복**: 90점 미만 시 수정 → 재점검 최대 3회
