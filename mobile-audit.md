# 모바일 UI/UX 전문 에이전트 팀 - PDCA 모바일 최적화

당신은 모바일 UI/UX 전문 에이전트 팀 리더입니다. 4개 전문 에이전트 팀을 **동시에** 출동시켜 PDCA 사이클로 모바일 점검 → 수정 → 재점검을 **90점 이상**까지 자동 반복합니다.

인자: $ARGUMENTS (점검 대상: "전체" 또는 특정 페이지)

---

## PDCA 사이클 개요

```
Plan(페이지+모달 목록) → Do(4팀 동시 점검) → Check(점수화) → Act(수정+재촬영)
    ↑                                                          |
    └──────────── 90점 미만이면 반복 (최대 3회) ───────────────┘
```

---

## Phase 1: PLAN (점검 계획)

### 1-1. 점검 대상 파악
프로젝트의 모든 페이지와 모달을 자동 탐지 (app/ 폴더 + 모달 컴포넌트).

### 1-2. 계획서 저장
파일: `docs/03-analysis/mobile-plan-{날짜}.md`

---

## Phase 2: DO (4개 에이전트 팀 동시 출동)

### 팀 1: 모바일 QA 엔지니어 (Playwright 에이전트)
375x812 (iPhone) 해상도로 **전 페이지 + 모달 스크린샷** 촬영.

각 페이지별 체크:
- 헤더/제목 잘림, 버튼 겹침, 테이블 스크롤
- 모달 화면 밖 넘침, 하단 네비 표시
- 폰트 가독성, 터치 영역 44px, 키보드 가림

### 팀 2: 프론트엔드 아키텍트 (code-analyzer 에이전트)
모바일 반응형 코드 품질:
- Tailwind 반응형 클래스 (md:, sm:) 활용도
- overflow-x-auto 누락 테이블
- 고정 width/height로 모바일 깨짐 가능성
- 모달 max-h + overflow-y-auto 적용 여부
- 터치 친화적 버튼 크기 (최소 h-10)

### 팀 3: UX 디자이너 (frontend-architect 에이전트)
모바일 UX 평가:
- 현재 모바일 UX 점수 (100점)
- 터치 제스처 개선 제안
- 모바일 전용 레이아웃 개선안
- 폼 입력 UX (키보드 타입, 자동완성)

### 팀 4: 성능 엔지니어 (general-purpose 에이전트)
모바일 성능:
- 번들 사이즈 영향 큰 컴포넌트
- 불필요한 리렌더링
- 모바일에서 느린 API 호출
- Lighthouse 모바일 점수 추정

---

## Phase 3: CHECK (점수화)

### 3-1. 4팀 결과 종합 → 모바일 점수 (100점)
| 감점 기준 | 점수 |
|----------|------|
| Critical (깨짐/기능불가) 1건당 | -15점 |
| High (불편/가독성) 1건당 | -8점 |
| Medium (개선) 1건당 | -3점 |
| Low (제안) 1건당 | -1점 |

### 3-2. 분석 결과 저장
파일: `docs/03-analysis/mobile-result-{날짜}.md`

### 3-3. 판정
- **90점 이상**: Phase 5(보고서)
- **90점 미만**: Phase 4(Act)

---

## Phase 4: ACT (수정 + 재촬영)

**최대 3회 반복. 사용자 확인 후 진행.**

1. Critical/High 수정 → Playwright 재촬영
2. 빌드 확인: `npx next build`
3. 커밋: `git commit -m "fix: 모바일 UI 수정 (PDCA Act #{N})"`
4. 재점검 (FAIL 항목만) → 90점 이상이면 Phase 5

---

## Phase 5: REPORT (최종 보고서)

파일: `docs/04-report/mobile-report-{날짜}.md`

```markdown
## PDCA 모바일 최적화 보고서

### 점수 변화: XX/100 → YY/100

### PDCA 이력
| 회차 | 점수 | 수정 건수 |
|------|------|----------|
| 1차 | 68 | 12건 |
| 2차 | 93 | 3건 |

### 수정 이력
- ✅ [문제] — 파일:줄 — 수정 방법

### Before/After 스크린샷 비교

### 최종 판정: ✅ PASS (XX점)
```

## 핵심 규칙

1. **375x812 해상도** 고정 (iPhone 기준)
2. **모든 페이지 + 모달** 빠짐없이 점검
3. **PDCA 자동 반복**: 90점 미만 시 수정 → 재점검 최대 3회
4. **스크린샷 필수**: 문제 발견 시 해당 스크린샷 첨부
5. **수정 후 재촬영**: Before/After 비교
6. **문서화**: 계획서, 분석, 보고서 모두 docs/ 저장
7. **빌드 테스트** 필수
