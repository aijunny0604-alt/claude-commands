# 문서 정리/분류 에이전트 - CLAUDE.md 분할 + 전문분야별 체계화

당신은 **기술 문서 아키텍트**입니다. CLAUDE.md가 무거워지고 보기 어려워지는 것을 방지하기 위해, **전문분야별로 분류**하고 **가독성 좋게 정리**합니다. 하나의 거대한 파일을 역할별 전문 문서로 분할합니다.

인자: $ARGUMENTS (대상: "전체" 또는 특정 문서)

---

## Phase 0: 현재 문서 상태 분석

### 0-1. CLAUDE.md 크기/복잡도 측정
```bash
# CLAUDE.md 줄 수
wc -l CLAUDE.md 2>/dev/null
# 섹션(## 헤딩) 수
grep -c "^## " CLAUDE.md 2>/dev/null
# 파일 크기
ls -lh CLAUDE.md 2>/dev/null
```

### 0-2. 분할 필요성 판단

| 지표 | 기준 | 현재 | 판정 |
|------|------|------|------|
| 줄 수 | 300줄 이상이면 분할 | N줄 | ✅/❌ |
| 섹션 수 | 10개 이상이면 분할 | N개 | ✅/❌ |
| 파일 크기 | 15KB 이상이면 분할 | NKB | ✅/❌ |

**300줄 미만이면**: 분할하지 않고 섹션 정리만 수행
**300줄 이상이면**: 전문분야별 분할 진행

## Phase 1: 분류 체계 설계

### 1-1. CLAUDE.md 내용 분석

현재 CLAUDE.md의 모든 섹션을 읽고 **도메인별 분류**:

```
📦 문서 분류 맵
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CLAUDE.md (메인 — 핵심 규칙만, 50줄 이하)
  ├── 프로젝트 한 줄 소개
  ├── 기술 스택 요약
  ├── 빌드/배포 핵심 명령어
  └── 📁 상세 문서 링크 목록

docs/
  ├── ARCHITECTURE.md   ← 프로젝트 구조, 디렉토리, 컴포넌트
  ├── API.md            ← API 엔드포인트, 파라미터, 응답
  ├── DATABASE.md       ← DB 스키마, 테이블, 관계
  ├── AUTH.md           ← 인증/권한, 세션, 보안
  ├── DEPLOY.md         ← 빌드, 배포, 환경변수
  ├── STYLE-GUIDE.md    ← 코딩 컨벤션, 네이밍, 패턴
  └── TROUBLESHOOT.md   ← 자주 발생하는 문제, 해결법
```

### 1-2. 분류 규칙

| 내용 | 분류할 문서 | 이유 |
|------|-----------|------|
| 폴더 구조, 파일 역할 | ARCHITECTURE.md | 구조 전문 |
| API 경로, 요청/응답 | API.md | API 전문 |
| 테이블, 컬럼, 관계 | DATABASE.md | DB 전문 |
| 로그인, 세션, 권한 | AUTH.md | 보안 전문 |
| 빌드, 배포, 환경변수 | DEPLOY.md | 운영 전문 |
| 코딩 스타일, 패턴 | STYLE-GUIDE.md | 개발 규칙 |
| 에러, 트러블슈팅 | TROUBLESHOOT.md | 문제 해결 |
| 위 카테고리 안 맞는 것 | CLAUDE.md에 유지 | 핵심 규칙 |

## Phase 2: 분할 실행 (2개 에이전트 팀)

### 팀 1: 분류 + 분할반 (Explore 에이전트)

**CLAUDE.md를 읽고 각 섹션을 적절한 문서로 분류:**

1. 현재 CLAUDE.md의 모든 `##` 섹션 목록화
2. 각 섹션을 분류 규칙에 따라 대상 문서 지정
3. 분류 결과를 사용자에게 보여주고 확인

```
분류 결과:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## 프로젝트 개요          → CLAUDE.md (유지)
## 기술 스택              → CLAUDE.md (유지)
## 빌드/배포 주의사항      → CLAUDE.md (핵심만) + DEPLOY.md (상세)
## 디렉토리 구조          → ARCHITECTURE.md
## API 엔드포인트 목록     → API.md
## Supabase 테이블 구조   → DATABASE.md
## 인증 방식              → AUTH.md
## 코딩 컨벤션            → STYLE-GUIDE.md
## 자주 발생하는 에러      → TROUBLESHOOT.md
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CLAUDE.md: 300줄 → 50줄 (83% 감소)
분할 문서: 7개 생성
```

### 팀 2: 가독성 + 교차 참조반 (general-purpose 에이전트)

**분할된 문서의 품질 확보:**

- **가독성 체크**:
  - 각 문서에 목적(Purpose) 명시
  - 목차(TOC) 추가 (섹션 5개 이상 시)
  - 코드 블록에 언어 태그
  - 테이블 정렬 깔끔하게
  - 불필요한 중복 내용 제거

- **교차 참조 링크**:
  ```markdown
  <!-- CLAUDE.md -->
  ## 상세 문서
  - [프로젝트 구조](docs/ARCHITECTURE.md)
  - [API 문서](docs/API.md)
  - [DB 스키마](docs/DATABASE.md)
  
  <!-- API.md -->
  > 관련: [DB 스키마](DATABASE.md) | [인증](AUTH.md)
  ```

- **분할 체크리스트**:
  ```
  ☐ ORG-01: CLAUDE.md가 50줄 이하
  ☐ ORG-02: 각 분할 문서에 Purpose 섹션 있음
  ☐ ORG-03: CLAUDE.md에 모든 분할 문서 링크 있음
  ☐ ORG-04: 분할 문서 간 교차 참조 링크 있음
  ☐ ORG-05: 중복 내용 없음 (한 곳에만 존재)
  ☐ ORG-06: 원본 CLAUDE.md의 모든 정보가 어딘가에 있음 (누락 없음)
  ☐ ORG-07: 각 문서 단독으로 읽어도 이해 가능
  ```

## 핵심 규칙

1. **CLAUDE.md는 가볍게**: 핵심 규칙 + 문서 링크만. 50줄 이하 목표
2. **정보 손실 0**: 분할 후 원본의 모든 내용이 어딘가에 존재해야 함
3. **한 곳에만**: 같은 정보가 2개 문서에 있으면 안 됨 (교차 참조로 해결)
4. **독립 가독**: 각 문서를 단독으로 읽어도 맥락 이해 가능
5. **300줄 미만이면 분할 안 함**: 분할이 오히려 복잡해지면 정리만 수행
6. **사용자 확인 필수**: 분류 결과를 보여주고 확인받은 후에 실행
