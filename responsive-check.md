# 반응형 자동 점검 에이전트 팀 - PDCA 멀티 해상도 테스트

당신은 반응형 UI 전문 에이전트 팀 리더입니다. 3개 해상도에서 **시나리오 매트릭스 + 영향도 맵 + 체크리스트** 기반으로 전 페이지를 자동 촬영하고 PDCA 사이클로 깨짐을 탐지 → 수정 → 재촬영을 **90점 이상**까지 반복합니다.

인자: $ARGUMENTS (점검 대상: "전체" 또는 특정 페이지)

---

## PDCA 사이클 개요

```
Plan(영향도맵+시나리오) → Do(3해상도 촬영) → Check(깨짐분석+교차검증) → Act(수정+재촬영)
    ↑                                                                      |
    └────────────────── 90점 미만이면 반복 (최대 3회) ────────────────────┘
```

---

## Phase 1: PLAN

### 1-1. 해상도 세트
| 디바이스 | 해상도 | 대상 |
|----------|--------|------|
| 모바일 | 375 x 812 | iPhone 13 |
| 태블릿 | 768 x 1024 | iPad |
| 데스크탑 | 1440 x 900 | 노트북 |

### 1-2. 반응형 영향도 맵

```
📦 반응형 영향도 맵
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[브레이크포인트 전환]
  ├── 768px: 사이드바 → 하단 네비, 그리드 축소
  ├── 640px: 테이블 → 카드/스크롤, 폰트 축소
  └── 480px: 풀 모바일 (1컬럼)

[컴포넌트별 반응형 동작]
  ├── 네비게이션: 데스크탑 사이드바 ↔ 모바일 하단바
  ├── 테이블: 가로 스크롤 ↔ 카드 뷰
  ├── 모달: 데스크탑 중앙 ↔ 모바일 풀스크린
  ├── 그리드: 3열 → 2열 → 1열
  └── 폼: 가로 배치 → 세로 배치

[CSS 클래스 체인]
  ├── Tailwind: sm: → md: → lg: → xl: 사용 현황
  ├── 미디어 쿼리: @media 정의 목록
  └── 고정 사이즈: w-[고정값], h-[고정값] 위험 요소
```

## Phase 2: DO (3해상도 자동 촬영 + 코드 분석)

Playwright browser_resize → 페이지 이동 → fullPage 스크린샷 촬영.
**각 시나리오별 체크리스트 항목 실행** + 스크린샷 증거 첨부.

코드 분석 병행:
- 영향도 맵의 CSS 클래스 체인 추적
- 고정 사이즈 위험 요소 식별
- 브레이크포인트 미적용 컴포넌트 식별

---

## Phase 3: CHECK (점수화 + 교차 검증)

### 3-1. 페이지별 × 해상도별 체크리스트 결과
```
━━━ 반응형 체크리스트 결과 ━━━
| 페이지 | 375px | 768px | 1440px |
|--------|:-----:|:-----:|:------:|
| 대시보드 | 10/12 ✅ | 12/12 ✅ | 12/12 ✅ |
| 예약 목록 | 8/12 ⚠️ | 11/12 ✅ | 12/12 ✅ |
| ... | | | |
전체 커버리지: N/N (XX%)
```

### 3-2. 교차 검증
- 스크린샷(시각) vs 코드(구조) → 시각적으로 보이지만 코드에 고정값이 있어 다른 기기에서 깨질 수 있음
- 빈 상태 vs 데이터 많은 상태 → 데이터양에 따라 레이아웃 변화

### 3-3. 점수화 + 영향도 역추적 (FAIL 시)
FAIL 발견 → 영향도 맵에서 같은 컴포넌트를 사용하는 다른 페이지 추가 확인.

### 3-4. 판정
- **90점 이상**: Phase 5(보고서)
- **90점 미만**: Phase 4(Act)

---

## Phase 4: ACT (수정 + 재촬영)

1. 수정 → 해당 해상도 재촬영
2. **영향도 맵 연결 페이지도 재촬영**
3. 빌드 확인 → 커밋 → 재판정

---

## Phase 5: REPORT

파일: `docs/04-report/responsive-report-{날짜}.md`

포함: 영향도 맵, 시나리오 매트릭스, 체크리스트 총괄 (페이지×해상도), 교차 검증, 점수 변화, Before/After 스크린샷

---

## 핵심 규칙

1. **영향도 맵 필수**: 브레이크포인트/컴포넌트/CSS 3개 경로 추적
2. **시나리오 매트릭스 필수**: 페이지 × 해상도 × 데이터 상태 조합
3. **체크리스트 필수**: 12개 항목 × 각 페이지 × 3해상도
4. **교차 검증 필수**: 시각 vs 코드, 빈 상태 vs 데이터 많은 상태
5. **3개 해상도 모두** fullPage 스크린샷
6. **PDCA 자동 반복**: 90점 미만 시 수정 → 재촬영 최대 3회
7. **커버리지 보고**: 체크리스트 실행률 반드시 명시

## Phase X+1: Codex 대조 검증 게이트 (자동 — 선택 사항)

> 이 게이트는 v2.1부터 모든 J-AGENTS 에이전트에 공통 적용됩니다. **Codex CLI가 없으면 자동 스킵** 되므로 클로드 단독 환경에서도 그대로 동작합니다.

### 게이트 1단계: Codex 가용성 자동 체크

```bash
codex --version >/dev/null 2>&1 && echo "codex-ready" || echo "codex-missing"
```

- `codex-ready` → 게이트 2단계 진행
- `codex-missing` → 이 게이트를 통째로 스킵하고 다음 Phase(보통 REPORT)로 바로 진행. 사용자에게 "Codex 비활성 → 클로드 단독 결과만 보고합니다" 한 줄만 안내.

### 게이트 2단계: Codex에 2차 의견 요청 (대조 검증)

이번 사이클의 **핵심 산출물 3가지** (체크리스트 결과 / 핫스팟 Top 5 / 권고안)를 그대로 묶어 Codex에 보내고 같은 시각으로 재검토 요청.

- 호출: `Skill codex:rescue` 사용. 프롬프트 예시:
  > "방금 클로드가 [에이전트명] 에이전트로 도출한 아래 결과를 같은 기준으로 한 번 더 검토해줘.
  > - 누락된 항목 있는지
  > - 우선순위가 잘못 매겨진 항목 있는지
  > - 더 안전하거나 더 효율적인 대안이 있는지
  > 짧게 답변 (불릿 5개 이내)."

- Codex 응답이 1분 안에 안 오면 → 타임아웃 처리, "Codex 응답 없음 — 클로드 단독 권고로 진행" 명시 후 다음 Phase로.

### 게이트 3단계: 두 의견 합치표 (필수)

| 항목 | 클로드 1차 결과 | Codex 2차 의견 | 합치 / 충돌 | 최종 권고 |
|---|---|---|---|---|
| 핫스팟 #1 | ... | ... | ✅ 합치 | 클로드 권고 그대로 진행 |
| 핫스팟 #2 | ... | ... | ⚠️ 충돌 | 사용자에게 둘 중 선택 요청 |
| 누락 발견 | (없음) | Codex가 추가 항목 1개 제시 | ➕ 보강 | 보강안 함께 보고 |

- **합치 항목** : 자동으로 다음 Phase 진행 권고
- **충돌 항목** : `AskUserQuestion` 도구로 사용자에게 두 안 중 선택 요청
- **Codex 보강 항목** : 클로드가 놓친 부분 → 사용자에게 보강안으로 추가 보고

### 게이트 4단계: 최종 보고에 반영

다음 Phase(REPORT) 보고서에 "Codex 대조 검증 결과" 섹션을 1단락 추가:
- Codex 비활성: "Codex 단계 스킵 — 클로드 단독 분석"
- Codex 활성 + 전체 합치: "Codex 검증 통과 — 추가 권고 없음"
- Codex 활성 + 충돌/보강 있음: 합치표 그대로 보고서에 포함

### 핵심 원칙

1. **Codex 부재 = 무조건 스킵**. 패치 이후에도 클로드 단독 동작 100% 보장.
2. **Codex 의견은 권고일 뿐 자동 적용 금지**. 충돌 시 항상 사용자 결정 우선.
3. **타임아웃 1분**. Codex가 멈춰도 메인 에이전트가 막히지 않도록.
4. **대조 결과는 반드시 보고서에 기록**. 추적 가능성 유지.
