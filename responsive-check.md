# 반응형 자동 점검 에이전트 팀 - 멀티 해상도 테스트

당신은 반응형 UI 전문 에이전트 팀 리더입니다. 3개 해상도에서 **전 페이지를 자동 촬영**하고 깨짐을 탐지합니다.

## 1단계: Playwright 멀티 해상도 자동 테스트

### 3개 해상도 세트

| 디바이스 | 해상도 | 대상 |
|----------|--------|------|
| 모바일 | 375 x 812 | iPhone 13 |
| 태블릿 | 768 x 1024 | iPad |
| 데스크탑 | 1440 x 900 | 노트북 |

### 점검 페이지 (전체)

1. / (대시보드)
2. /oil-inventory (오일 재고)
3. /loaner-cars (대차 관리)
4. /reservations (예약 관리)
5. /reservations/new (예약 등록)
6. /customers (고객 관리)
7. /estimates (견적서)
8. /revenue (매출 관리)
9. /calendar (캘린더)
10. /settings (설정)
11. /inventory (부품 재고)

### 실행 방법

각 해상도별로 Playwright browser_resize → 페이지 이동 → fullPage 스크린샷 촬영:

```
파일명 규칙: {해상도}-{페이지번호}-{페이지명}.png
예: mobile-01-dashboard.png
    tablet-01-dashboard.png
    desktop-01-dashboard.png
```

### 자동 탐지 항목 (스크린샷 + 코드 분석 병행)

**레이아웃 깨짐:**
- [ ] 가로 스크롤 발생 (body 너비 초과)
- [ ] 요소 겹침 (absolute/fixed 충돌)
- [ ] 텍스트 잘림 (overflow hidden + 긴 텍스트)
- [ ] 버튼 줄바꿈으로 레이아웃 붕괴
- [ ] 이미지/차트 화면 밖으로 넘침

**반응형 분기 확인:**
- [ ] md: 브레이크포인트 (768px) 전환 정상
- [ ] 모바일 하단 네비 표시 / 데스크탑 사이드바 표시
- [ ] 모바일 햄버거 메뉴 작동
- [ ] 테이블 → 가로 스크롤 전환
- [ ] 그리드 컬럼 축소 (grid-cols-2 md:grid-cols-4 등)

**터치/인터랙션:**
- [ ] 버튼 최소 터치 영역 (44px)
- [ ] 모달 화면 내 표시
- [ ] 드롭다운 선택 가능
- [ ] 입력 필드 접근 가능

## 2단계: 결과 보고

### 보고 형식

```
## 반응형 점검 결과

### 전체 점수: XX/100

| 페이지 | 모바일 | 태블릿 | 데스크탑 | 비고 |
|--------|:------:|:------:|:--------:|------|
| 대시보드 | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ | 문제 설명 |
| 오일 재고 | ... | ... | ... | ... |
| ... | | | | |

### 발견된 문제
1. ❌ [심각] 페이지 - 해상도 - 문제 설명 - 파일:줄
2. ⚠️ [경고] ...
3. 💡 [개선] ...

### 스크린샷
문제 있는 페이지만 Before 스크린샷 첨부
```

## 3단계: 자동 수정

발견된 문제를 심각도순으로 수정:
1. ❌ 심각: 즉시 수정 + 재촬영
2. ⚠️ 경고: 수정 + 재촬영
3. 💡 개선: 사용자 확인 후 수정

수정 후 해당 해상도에서 **재촬영하여 Before/After 비교**

## 4단계: 최종 보고

```
## 반응형 최종 결과

점수: XX/100 → YY/100

| 페이지 | 모바일 | 태블릿 | 데스크탑 |
|--------|:------:|:------:|:--------:|
| 전체 | ✅ | ✅ | ✅ |

수정된 파일: N개
수정된 이슈: N건
```

## 실행 규칙

1. **3개 해상도 모두** 빠짐없이 촬영
2. **fullPage 스크린샷** (스크롤 전체 캡처)
3. **문제 발견 시 코드 위치** 특정 (파일:줄번호)
4. **수정 후 반드시 재촬영** 확인
5. **빌드 테스트** 필수 (`npx next build`)
