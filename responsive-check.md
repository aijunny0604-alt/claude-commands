# 반응형 자동 점검 에이전트 팀 - PDCA 멀티 해상도 테스트

당신은 반응형 UI 전문 에이전트 팀 리더입니다. 3개 해상도에서 **전 페이지를 자동 촬영**하고 PDCA 사이클로 깨짐을 탐지 → 수정 → 재촬영을 **90점 이상**까지 반복합니다.

인자: $ARGUMENTS (점검 대상: "전체" 또는 특정 페이지)

---

## PDCA 사이클 개요

```
Plan(해상도+페이지 목록) → Do(3해상도 촬영) → Check(깨짐 분석+점수) → Act(수정+재촬영)
    ↑                                                                   |
    └────────────────── 90점 미만이면 반복 (최대 3회) ─────────────────┘
```

---

## Phase 1: PLAN (점검 계획)

### 1-1. 해상도 세트
| 디바이스 | 해상도 | 대상 |
|----------|--------|------|
| 모바일 | 375 x 812 | iPhone 13 |
| 태블릿 | 768 x 1024 | iPad |
| 데스크탑 | 1440 x 900 | 노트북 |

### 1-2. 점검 페이지
프로젝트의 모든 페이지를 자동 탐지 (app/ 폴더 기준).

### 1-3. 계획서 저장
파일: `docs/03-analysis/responsive-plan-{날짜}.md`

---

## Phase 2: DO (3해상도 자동 촬영)

Playwright browser_resize → 페이지 이동 → fullPage 스크린샷 촬영.

### 자동 탐지 항목 (스크린샷 + 코드 분석 병행)

**레이아웃 깨짐:**
- 가로 스크롤 발생 (body 너비 초과)
- 요소 겹침 (absolute/fixed 충돌)
- 텍스트 잘림 (overflow hidden + 긴 텍스트)
- 버튼 줄바꿈으로 레이아웃 붕괴
- 이미지/차트 화면 밖으로 넘침

**반응형 분기 확인:**
- md: 브레이크포인트 (768px) 전환 정상
- 모바일 네비 / 데스크탑 사이드바 전환
- 테이블 → 가로 스크롤 전환
- 그리드 컬럼 축소

**터치/인터랙션:**
- 버튼 최소 터치 영역 (44px)
- 모달 화면 내 표시
- 드롭다운/입력 필드 접근 가능

---

## Phase 3: CHECK (점수화)

### 3-1. 페이지별 점수표
| 페이지 | 모바일 | 태블릿 | 데스크탑 | 비고 |
|--------|:------:|:------:|:--------:|------|
| 각 페이지 | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ | 문제 설명 |

### 3-2. 전체 점수 (100점)
| 감점 기준 | 점수 |
|----------|------|
| ❌ 심각 (깨짐) 1건당 | -10점 |
| ⚠️ 경고 (불편) 1건당 | -3점 |
| 💡 개선 1건당 | -1점 |

### 3-3. 분석 결과 저장
파일: `docs/03-analysis/responsive-result-{날짜}.md`

### 3-4. 판정
- **90점 이상**: Phase 5(보고서)
- **90점 미만**: Phase 4(Act)

---

## Phase 4: ACT (수정 + 재촬영)

**최대 3회 반복.**

1. ❌ 심각 → 즉시 수정 + 해당 해상도 재촬영
2. ⚠️ 경고 → 수정 + 재촬영
3. 빌드 확인: `npx next build`
4. 커밋: `git commit -m "fix: 반응형 깨짐 수정 (PDCA Act #{N})"`
5. 재판정 → 90점 이상이면 Phase 5

---

## Phase 5: REPORT (최종 보고서)

파일: `docs/04-report/responsive-report-{날짜}.md`

```markdown
## PDCA 반응형 점검 보고서

### 점수 변화: XX/100 → YY/100

| 페이지 | 모바일 | 태블릿 | 데스크탑 |
|--------|:------:|:------:|:--------:|
| 전체 | ✅ | ✅ | ✅ |

### PDCA 이력
| 회차 | 점수 | 수정 건수 |
|------|------|----------|
| 1차 | 72 | 8건 |
| 2차 | 95 | 2건 |

### 수정된 파일: N개
### Before/After 비교: [스크린샷 포함]
### 최종 판정: ✅ PASS (XX점)
```

## 핵심 규칙

1. **3개 해상도 모두** 빠짐없이 촬영
2. **fullPage 스크린샷** (스크롤 전체 캡처)
3. **PDCA 자동 반복**: 90점 미만 시 수정 → 재촬영 최대 3회
4. **수정 후 반드시 재촬영** Before/After 비교
5. **빌드 테스트** 필수
6. **문서화**: 계획서, 분석, 보고서 모두 docs/ 저장
