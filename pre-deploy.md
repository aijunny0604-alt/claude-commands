# 배포 전 자동 체크리스트 - PDCA 배포 게이트

배포 전 필수 점검을 **영향도 맵 + 체크리스트** 기반으로 자동 수행합니다. 모든 항목 PASS여야 배포를 허용합니다.

인자: $ARGUMENTS ("local" 또는 "production")

---

## Phase 0: 배포 영향도 맵 생성

### 0-1. 변경 범위 파악
```bash
git log --oneline -5
git diff --stat HEAD~1
git diff --name-only HEAD~1
```

### 0-2. 배포 영향도 맵

```
📦 배포 영향도 맵
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[변경 파일] → [빌드 영향] → [런타임 영향]
  ├── API 변경 → 기존 클라이언트 호환성
  ├── DB 스키마 변경 → 마이그레이션 필요
  ├── 환경변수 변경 → 프로덕션 설정 확인
  ├── 의존성 변경 → package-lock 동기화
  └── UI 변경 → 캐시 무효화 필요
```

## Phase 1: 배포 체크리스트 자동 실행

```
━━━ 배포 마스터 체크리스트 ━━━
[빌드] (자동)
  ☐ DEP-01: npx next build 성공
  ☐ DEP-02: 빌드 경고 0건 (치명적)
  ☐ DEP-03: 번들 사이즈 증가 확인

[타입] (자동)
  ☐ DEP-04: npx tsc --noEmit 성공
  ☐ DEP-05: 타입 에러 0건

[코드] (자동)
  ☐ DEP-06: 미커밋 변경사항 없음
  ☐ DEP-07: console.log/debugger 잔류 없음
  ☐ DEP-08: TODO/FIXME 신규 추가 없음

[환경] (자동)
  ☐ DEP-09: .env.local 존재
  ☐ DEP-10: 필수 환경변수 누락 없음
  ☐ DEP-11: 하드코딩 시크릿 없음

[DB] (자동)
  ☐ DEP-12: npx prisma validate 성공
  ☐ DEP-13: 미적용 마이그레이션 없음
  ☐ DEP-14: 스키마-DB 동기화 상태

[영향도] (영향도 맵 기반)
  ☐ DEP-15: 변경된 API의 클라이언트 호환성 확인
  ☐ DEP-16: 변경된 DB 필드의 기존 데이터 영향 확인
  ☐ DEP-17: 새 환경변수가 프로덕션에 설정되어 있는지

[프로덕션] (production일 때만)
  ☐ DEP-18: 프로덕션 헬스체크 200 응답
  ☐ DEP-19: 최근 커밋 위험도 평가 (변경 파일 수 + 유형)

[UI 스모크 테스트] (playwright-cli 사용)
  ☐ DEP-20: 로그인 페이지 로드 → 로그인 성공
  ☐ DEP-21: 메인 페이지 데이터 표시 정상
  ☐ DEP-22: 핵심 기능 1개 클릭 → 동작 확인 (모달 열림 등)
  ☐ DEP-23: 콘솔 에러 0건
```

## Phase 2: 체크리스트 실행

각 항목을 순서대로 실행하고 결과 기록:

```bash
# DEP-01: 빌드
npx next build

# DEP-04: 타입
npx tsc --noEmit

# DEP-06: 미커밋
git status
git diff --stat

# DEP-09: 환경변수
test -f .env.local && echo "OK" || echo "MISSING"

# DEP-12: DB
npx prisma validate

# DEP-18: 프로덕션 헬스 (production 모드)
curl -s -o /dev/null -w "%{http_code}" {프로덕션URL}
```

## Phase 4: 문서 최신화 (점검 PASS 시 자동 실행)

**체크리스트 전체 PASS 판정 후**, 배포 전 문서를 최신 상태로 정리합니다.

### 4-1. 문서-코드 불일치 스캔

변경된 코드를 기반으로 **문서가 뒤처진 부분** 자동 탐지:

```
━━━ 문서 최신화 체크리스트 ━━━
[CLAUDE.md]
  ☐ SYNC-01: 새 API 추가됐으면 API 섹션에 반영
  ☐ SYNC-02: 새 환경변수 추가됐으면 환경변수 섹션에 반영
  ☐ SYNC-03: 디렉토리 구조 변경됐으면 구조 섹션 업데이트
  ☐ SYNC-04: 의존성 추가/삭제됐으면 기술 스택 업데이트
  ☐ SYNC-05: 빌드/배포 방법 변경됐으면 해당 섹션 업데이트

[README.md]
  ☐ SYNC-06: 새 기능 추가됐으면 기능 목록 업데이트
  ☐ SYNC-07: 설치/실행 방법 변경됐으면 업데이트

[기타 문서] (docs/ 폴더 존재 시)
  ☐ SYNC-08: 분할 문서(ARCHITECTURE, API, DB 등) 해당 내용 업데이트
  ☐ SYNC-09: 교차 참조 링크 깨진 것 없음

[pre-deploy 자체]
  ☐ SYNC-10: 이번 변경으로 체크리스트에 항목 추가/수정 필요한지 확인
```

### 4-2. 자동 수정

불일치 발견 시 **즉시 문서 수정**:
- 새 API → CLAUDE.md에 엔드포인트 추가
- 새 환경변수 → CLAUDE.md + .env.example에 추가
- 구조 변경 → 디렉토리 트리 재생성
- 기능 추가 → README 기능 목록 업데이트

### 4-3. 수정 후 재검증

수정한 문서를 다시 체크:
```
문서 최신화 결과:
  ✅ SYNC-01: /api/vehicles 추가 반영 완료
  ✅ SYNC-02: 환경변수 변경 없음 (스킵)
  ✅ SYNC-03: 구조 변경 없음 (스킵)
  ⚠️ SYNC-04: dayjs 의존성 추가 → CLAUDE.md 기술 스택에 반영 완료
  ✅ SYNC-05~10: 변경 없음

  최신화율: 10/10 (100%)
```

### 4-4. 문서 변경 커밋

문서 수정이 있었으면 별도 커밋:
```bash
git add CLAUDE.md README.md docs/ .env.example 2>/dev/null
git commit -m "docs: 배포 전 문서 최신화 (pre-deploy sync)"
```

## 핵심 규칙

1. **영향도 맵 필수**: 변경 파일 기반 배포 영향 사전 파악
2. **체크리스트 23개 전부 실행**: 건너뛰기 금지
3. **빌드 실패 시 즉시 중단**: 자동 수정 시도 최대 3회
4. **점검 PASS 후 문서 최신화 필수**: 코드만 배포하고 문서가 뒤처지면 안 됨
5. **문서 수정 시 별도 커밋**: 코드 커밋과 분리
6. **커버리지 보고**: 코드 체크리스트 + 문서 체크리스트 통과율 모두 명시
