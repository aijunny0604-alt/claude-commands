# 배포 전 자동 체크리스트 - PDCA 배포 게이트

배포 전 필수 점검을 **영향도 맵 + 체크리스트** 기반으로 자동 수행합니다. 모든 항목 PASS여야 배포를 허용합니다.

인자: $ARGUMENTS ("local" 또는 "production")

---

## Phase 0: 배포 영향도 맵 생성

### 0-1. 변경 범위 파악
```bash
git log --oneline -5
git diff --stat HEAD~1
git diff --name-only HEAD~1
```

### 0-2. 배포 영향도 맵

```
📦 배포 영향도 맵
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[변경 파일] → [빌드 영향] → [런타임 영향]
  ├── API 변경 → 기존 클라이언트 호환성
  ├── DB 스키마 변경 → 마이그레이션 필요
  ├── 환경변수 변경 → 프로덕션 설정 확인
  ├── 의존성 변경 → package-lock 동기화
  └── UI 변경 → 캐시 무효화 필요
```

## Phase 1: 배포 체크리스트 자동 실행

```
━━━ 배포 마스터 체크리스트 ━━━
[빌드] (자동)
  ☐ DEP-01: npx next build 성공
  ☐ DEP-02: 빌드 경고 0건 (치명적)
  ☐ DEP-03: 번들 사이즈 증가 확인

[타입] (자동)
  ☐ DEP-04: npx tsc --noEmit 성공
  ☐ DEP-05: 타입 에러 0건

[코드] (자동)
  ☐ DEP-06: 미커밋 변경사항 없음
  ☐ DEP-07: console.log/debugger 잔류 없음
  ☐ DEP-08: TODO/FIXME 신규 추가 없음

[환경] (자동)
  ☐ DEP-09: .env.local 존재
  ☐ DEP-10: 필수 환경변수 누락 없음
  ☐ DEP-11: 하드코딩 시크릿 없음

[DB] (자동)
  ☐ DEP-12: npx prisma validate 성공
  ☐ DEP-13: 미적용 마이그레이션 없음
  ☐ DEP-14: 스키마-DB 동기화 상태

[영향도] (영향도 맵 기반)
  ☐ DEP-15: 변경된 API의 클라이언트 호환성 확인
  ☐ DEP-16: 변경된 DB 필드의 기존 데이터 영향 확인
  ☐ DEP-17: 새 환경변수가 프로덕션에 설정되어 있는지

[프로덕션] (production일 때만)
  ☐ DEP-18: 프로덕션 헬스체크 200 응답
  ☐ DEP-19: 최근 커밋 위험도 평가 (변경 파일 수 + 유형)

[UI 스모크 테스트] (playwright-cli 사용)
  ☐ DEP-20: 로그인 페이지 로드 → 로그인 성공
  ☐ DEP-21: 메인 페이지 데이터 표시 정상
  ☐ DEP-22: 핵심 기능 1개 클릭 → 동작 확인 (모달 열림 등)
  ☐ DEP-23: 콘솔 에러 0건
```

## Phase 2: 체크리스트 실행

각 항목을 순서대로 실행하고 결과 기록:

```bash
# DEP-01: 빌드
npx next build

# DEP-04: 타입
npx tsc --noEmit

# DEP-06: 미커밋
git status
git diff --stat

# DEP-09: 환경변수
test -f .env.local && echo "OK" || echo "MISSING"

# DEP-12: DB
npx prisma validate

# DEP-18: 프로덕션 헬스 (production 모드)
curl -s -o /dev/null -w "%{http_code}" {프로덕션URL}
```

❌가 하나라도 있으면 자동 수정 시도 → 재체크. 수정 불가 시 사용자에게 보고.

## Phase 4: 배포 실행

Phase 1~3 모두 PASS 시 자동 배포:

```bash
# 변경사항 커밋 (있는 경우)
git add <변경파일>
git commit -m "feat/fix: 설명 [deploy]"
git push origin master
```

**[deploy] 태그 필수**: Vercel은 `[deploy]` 키워드가 있는 커밋만 배포합니다.

---

## Phase 5: 배포 후 실전 검증 (필수 — 스킵 금지)

**배포 성공 != 실제 작동**. Vercel 빌드 완료 후 **프로덕션 URL에서 직접 Playwright로 검증**합니다.

### 5-1. 배포 대기 + 헬스체크
```bash
# Vercel 빌드 대기 (일반적으로 30~60초)
sleep 45

# 프로덕션 헬스체크
curl -s -o /dev/null -w "%{http_code}" https://{프로덕션URL}/login
# 200 OK 확인
```

### 5-2. 프로덕션 UI 실전 검증 (Playwright 필수)

```
[POST-01] 프로덕션 로그인
  - browser_navigate → https://{프로덕션URL}/login
  - browser_fill_form → 비밀번호 입력
  - browser_click → 로그인 버튼
  - 리디렉션 확인

[POST-02] 변경된 기능 실제 렌더 확인 (영향도 맵 기반)
  - 변경 파일이 페이지 UI를 수정한 경우:
    - 해당 페이지로 navigate
    - 변경된 요소가 실제로 보이는지 snapshot 확인
    - 스크린샷 캡처 (증거)
  - 변경 파일이 API인 경우:
    - 해당 API를 호출하는 페이지로 navigate
    - 데이터가 정상 표시되는지 확인

[POST-03] 콘솔 에러 스캔
  - browser_console_messages
  - error 레벨 0건 확인 (favicon.ico 404 제외)

[POST-04] 기존 기능 회귀 체크 (최소 3개 페이지)
  - /, /reservations, /customers 등 주요 페이지
  - 각 페이지 로드 + 콘솔 에러 0건

[POST-05] 배포 후 증거 스크린샷 저장
```

### 5-3. Phase 5 체크리스트

```
━━━ 배포 후 검증 체크리스트 ━━━
  ☐ POST-01: 프로덕션 로그인 성공
  ☐ POST-02: 변경된 기능 프로덕션에서 실제 작동 확인
  ☐ POST-03: 콘솔 에러 0건 (앱 에러)
  ☐ POST-04: 기존 페이지 3개 이상 회귀 없음
  ☐ POST-05: 프로덕션 스크린샷 캡처 (증거)
```

### 5-4. 결과 보고

```
## 배포 후 검증 결과

| # | 항목 | 결과 | 비고 |
|---|------|------|------|
| POST-01 | 프로덕션 로그인 | ✅ | |
| POST-02 | 변경 기능 작동 | ✅ | 스크린샷: xxx.png |
| POST-03 | 콘솔 에러 | ✅ | 0건 |
| POST-04 | 회귀 체크 | ✅ | 3/3 페이지 정상 |
| POST-05 | 스크린샷 | ✅ | |

### 최종 판정: ✅ 배포 성공 및 작동 확인 / ❌ 롤백 필요
```

**POST-XX 중 하나라도 FAIL이면:**
- 🔴 Critical: 즉시 수정 → 재배포
- 🟡 Warning: 원인 파악 후 사용자에게 보고
- 이전 커밋으로 롤백 권장 (`git revert` + push [deploy])

## 핵심 규칙

1. **영향도 맵 필수**: 변경 파일 기반 배포 영향 사전 파악
2. **체크리스트 19개 전부 실행**: 건너뛰기 금지
3. **빌드 실패 시 즉시 중단**: 자동 수정 시도 최대 3회
4. **커버리지 보고**: 체크리스트 실행률 반드시 명시
5. **배포 후 검증 필수 (Phase 5)**: HTTP 200만으로 끝내지 말 것 — Playwright로 실제 UI 확인
6. **증거 기반 PASS**: 스크린샷/로그 없는 PASS는 무효
7. **로컬 검증 ≠ 프로덕션 검증**: 로컬 PASS해도 프로덕션에서 다시 확인
