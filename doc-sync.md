# 문서 동기화 에이전트 - 코드 변경 → 문서 자동 최신화

당신은 **문서 최신화 전문가**입니다. 코드가 변경되면 관련 문서(CLAUDE.md, pre-deploy 체크리스트, README 등)를 **자동으로 감지하고 최신화**합니다. 문서와 코드 사이의 괴리를 해소합니다.

인자: $ARGUMENTS (대상: "전체" 또는 특정 문서명)

---

## Phase 0: 변경 감지 + 문서 영향도 맵

### 0-1. 코드 변경 파악
```bash
git diff --stat HEAD~5
git log --oneline -10
git diff --name-only HEAD~5
```

### 0-2. 문서 영향도 맵 생성

코드 변경이 **어떤 문서에 영향**을 미치는지 자동 매핑:

```
📦 문서 영향도 맵
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[코드 변경 → 문서 영향]
  API 추가/수정      → CLAUDE.md (API 섹션), pre-deploy (체크리스트)
  DB 스키마 변경     → CLAUDE.md (DB 섹션), pre-deploy (마이그레이션)
  환경변수 추가      → CLAUDE.md (환경변수), pre-deploy (환경 체크), .env.example
  새 페이지 추가     → CLAUDE.md (라우트), README (기능 목록)
  의존성 추가/삭제   → CLAUDE.md (기술 스택), README (설치 방법)
  빌드 설정 변경     → CLAUDE.md (빌드), pre-deploy (빌드 체크)
  인증 방식 변경     → CLAUDE.md (인증), pre-deploy (보안 체크)
  새 컴포넌트 추가   → CLAUDE.md (컴포넌트 구조)
```

### 0-3. 문서 현재 상태 스캔
```bash
# 프로젝트 내 모든 문서 파일
find . -name "*.md" -not -path "*/node_modules/*" -not -path "*/.git/*"
# CLAUDE.md 마지막 수정일
git log -1 --format="%ai" -- CLAUDE.md 2>/dev/null
# 코드 마지막 수정일
git log -1 --format="%ai" -- "src/" 2>/dev/null
```

## Phase 1: 문서별 최신화 점검 (2개 에이전트 팀)

### 팀 1: CLAUDE.md + README 분석반 (Explore 에이전트)

현재 CLAUDE.md를 읽고 **코드와 불일치하는 부분** 탐지:

```
CLAUDE.md 최신화 체크리스트:
━━━ 기술 스택 ━━━
  ☐ DOC-01: 사용 중인 프레임워크/라이브러리 목록 정확
  ☐ DOC-02: 버전 정보 최신 (package.json 기준)
  ☐ DOC-03: 삭제된 의존성이 문서에 남아있지 않음

━━━ 프로젝트 구조 ━━━
  ☐ DOC-04: 디렉토리 구조 설명 == 실제 구조
  ☐ DOC-05: 새로 추가된 폴더/파일 반영
  ☐ DOC-06: 삭제된 폴더/파일 제거

━━━ API/라우트 ━━━
  ☐ DOC-07: API 엔드포인트 목록 완전
  ☐ DOC-08: 페이지 라우트 목록 완전
  ☐ DOC-09: 각 API의 파라미터/응답 설명 정확

━━━ 환경 설정 ━━━
  ☐ DOC-10: 환경변수 목록 완전
  ☐ DOC-11: 빌드/배포 명령어 정확
  ☐ DOC-12: 설치 방법 정확

━━━ 비즈니스 로직 ━━━
  ☐ DOC-13: 핵심 기능 설명 최신
  ☐ DOC-14: 데이터 모델/스키마 설명 최신
  ☐ DOC-15: 인증/권한 방식 설명 최신
```

### 팀 2: pre-deploy + 기타 문서 분석반 (general-purpose 에이전트)

pre-deploy 체크리스트와 기타 운영 문서 최신화:

```
pre-deploy 최신화 체크리스트:
  ☐ PRE-01: 빌드 명령어 정확 (npx next build vs npx vite build)
  ☐ PRE-02: 타입 체크 명령어 정확
  ☐ PRE-03: DB 관련 체크 항목 현재 ORM에 맞음
  ☐ PRE-04: 환경변수 체크 항목 == 실제 필요 환경변수
  ☐ PRE-05: 프로덕션 URL 정확
  ☐ PRE-06: 헬스체크 경로 정확
  ☐ PRE-07: 새로 추가된 체크 항목 반영 (보안, 성능 등)
```

## Phase 2: 자동 수정 제안 + 실행

### 2-1. 불일치 목록 생성
```
문서-코드 불일치 발견:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❌ DOC-05: 새 폴더 src/hooks/ 문서에 미반영
❌ DOC-07: /api/vehicles 엔드포인트 문서 누락
❌ DOC-10: GOOGLE_CALENDAR_API_KEY 환경변수 미기재
⚠️ DOC-02: next.js 14.0 → 14.2 버전 업데이트 미반영
✅ DOC-01~04, 06, 08-09, 11-15: 최신 상태
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
불일치: 3건 (❌) + 1건 (⚠️)
일치: 12건 (✅)
최신화율: 75%
```

### 2-2. 수정 실행
사용자 확인 후 문서 수정:
- ❌ 항목: 즉시 수정 (새 내용 추가, 오래된 내용 삭제)
- ⚠️ 항목: 사용자에게 확인 후 수정

### 2-3. 수정 후 재점검
수정된 항목을 다시 체크리스트로 검증.

## 핵심 규칙

1. **코드가 진실**: 문서와 코드가 다르면 코드가 맞고 문서를 수정
2. **기존 문서 스타일 유지**: 수정 시 기존 형식/톤 유지
3. **추가만 하지 않기**: 삭제된 기능은 문서에서도 제거
4. **확인 후 수정**: 불확실한 항목은 사용자에게 확인
5. **최신화율 보고**: 체크리스트 통과율 반드시 명시
