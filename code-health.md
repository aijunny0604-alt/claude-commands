# 코드 품질 점검 에이전트 팀 - PDCA 기술 부채 관리

당신은 코드 품질 전문 에이전트 팀 리더입니다. 3개 전문 에이전트 팀을 **동시에** 출동시켜 **시나리오 매트릭스 + 영향도 맵 + 체크리스트** 기반 PDCA 사이클로 코드 품질 점검 → 리팩토링 → 재점검을 **90점 이상**까지 자동 반복합니다.

인자: $ARGUMENTS (점검 대상: "전체" 또는 특정 디렉토리/기능)

---

## Phase 1: PLAN

### 1-1. 코드 현황
```bash
find src -name "*.ts" -o -name "*.tsx" | xargs wc -l 2>/dev/null | tail -1
find src -name "*.ts" -o -name "*.tsx" | wc -l
git log --pretty=format: --name-only --since="1 month ago" | sort | uniq -c | sort -rn | head -20
```

### 1-2. 코드 의존성 영향도 맵

```
📦 코드 의존성 영향도 맵
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[파일 의존성 트리]
  ├── 핵심 파일 (import 10개+): 수정 시 광범위 영향
  ├── 순환 참조: A→B→A (있으면 Critical)
  └── 고아 파일: 어디서도 import 안 됨

[패턴 공유 맵]
  ├── 같은 CRUD 패턴: API 핸들러 N개
  ├── 같은 에러 핸들링: try/catch 패턴 N개
  ├── 같은 폼 처리: 커스텀 훅 가능 N개
  └── 같은 모달 패턴: 공통화 가능 N개

[복잡도 핫스팟]
  ├── 100줄+ 함수 (Top 5)
  ├── 300줄+ 컴포넌트 (Top 5)
  ├── useState 10개+ 컴포넌트
  └── useEffect 5개+ 컴포넌트
```

### 1-3. 코드 품질 시나리오 매트릭스

```
X축 (품질 영역): 중복 | 복잡도 | 미사용 | 일관성 | 타입안전성
Y축 (코드 유형): 컴포넌트 | API | 유틸 | 타입 | 스타일
Z축 (리스크):   안전(삭제) | 중위험(공통화) | 고위험(분리)

도출 (최소 20개):
  CQ-01: 10줄+ 동일 코드 블록 탐지
  CQ-02: 유사 API 핸들러 패턴 공통화 가능?
  CQ-03: God Component(500줄+) 분리 필요?
  CQ-04: 미사용 export 삭제 안전성
  ...
```

```

[복잡도] (팀2)

  ☐ UN-07: 매직 넘버/스트링
```

## Phase 2: DO (3팀 동시 출동)

### 팀 1: 중복 탐지반 — DUP 체크리스트 담당
영향도 맵의 "패턴 공유 맵"을 따라 중복 코드 + 공통화 기회 분석.

### 팀 2: 복잡도 분석반 — CX 체크리스트 담당
영향도 맵의 "복잡도 핫스팟"을 따라 구조 분석.

### 팀 3: 미사용 코드 + 의존성 점검반 — UN 체크리스트 담당
Dead code + 의존성 분석.

## Phase 3: CHECK (점수화 + 교차 검증)

### 3-1. 체크리스트 결과 취합 + 커버리지
### 3-2. 교차 검증
- 팀1(중복 발견) + 팀2(복잡도 높음) → **공통화로 복잡도도 줄일 수 있음**
- 팀2(큰 컴포넌트) + 팀3(미사용 코드) → **미사용 코드 제거 후 크기 확인**

### 3-3. 영향도 맵 기반 리팩토링 리스크 평가
```

  → 리스크: 중 (기능 테스트 필요)
  → 권장: 공통 유틸 추출 후 /change-verify 실행
```

## Phase 4: ACT (안전한 순서로 리팩토링)

1. 미사용 코드 삭제 (안전)
2. 매직 넘버 상수화 (안전)
3. 중복 코드 공통화 (중위험 — 영향도 맵 확인)
4. 복잡 함수 분리 (중위험 — 영향도 맵 확인)
5. **각 수정 후 영향도 맵 연결 파일 검증**

빌드 확인 → 커밋 → 재점검. 최대 3회 반복.

## 핵심 규칙

1. **영향도 맵 필수**: 의존성/패턴/핫스팟 3가지 추적
2. **시나리오 매트릭스 필수**: 품질 영역 × 코드 유형 × 리스크 조합
3. **체크리스트 필수**: 17개 항목 전부 실행 + 결과 기록
4. **교차 검증 필수**: 중복+복잡도+미사용 결과 교차 분석
5. **안전한 순서**: 삭제 → 상수화 → 공통화 → 분리
6. **PDCA 자동 반복**: 90점 미만 시 리팩토링 → 재점검 최대 3회
